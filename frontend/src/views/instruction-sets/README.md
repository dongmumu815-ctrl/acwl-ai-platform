# 指令集管理模块

## 概述

指令集管理模块是ACWL-AI大模型管理平台的核心功能之一，用于创建、管理和维护树状结构的指令集，主要应用于内容审核和自动化处理场景。

## 功能特性

### 📋 指令集管理
- **创建指令集**: 支持创建新的指令集，包含名称、描述、版本等基本信息
- **状态管理**: 支持草稿(DRAFT)、活跃(ACTIVE)、未激活(INACTIVE)三种状态
- **版本控制**: 支持指令集版本管理，便于迭代和回滚
- **树状结构**: 支持复杂的树状指令集结构，满足多层级处理需求

### 📊 统计分析
- **实时统计**: 显示总指令集数量、活跃指令集、草稿状态、总执行次数
- **执行统计**: 提供详细的执行统计信息，包括成功次数、平均执行时间等
- **性能监控**: 监控指令集的执行性能和使用情况

### 🔍 搜索与筛选
- **全文搜索**: 支持按名称或描述搜索指令集
- **状态筛选**: 可按状态筛选指令集
- **高级筛选**: 支持多条件组合筛选

### 🧪 测试功能
- **在线测试**: 提供专门的测试页面，支持实时测试指令集
- **调试模式**: 支持正常模式和调试模式两种测试方式
- **超时控制**: 可配置测试超时时间(1-300秒)
- **结果展示**: 实时显示测试结果和执行详情

## 技术实现

### 前端技术栈
- **Vue 3**: 使用Composition API构建响应式界面
- **Element Plus**: UI组件库，提供丰富的交互组件
- **TypeScript**: 类型安全的JavaScript超集
- **Pinia**: 状态管理库

### 核心功能实现

#### 状态管理
```typescript
// 指令集状态枚举
enum InstructionSetStatus {
  DRAFT = 'draft',      // 草稿
  ACTIVE = 'active',    // 活跃
  INACTIVE = 'inactive' // 未激活
}
```

#### API接口
- `GET /api/instruction-sets`: 获取指令集列表
- `POST /api/instruction-sets`: 创建指令集
- `PUT /api/instruction-sets/:id`: 更新指令集
- `DELETE /api/instruction-sets/:id`: 删除指令集
- `POST /api/instruction-sets/:id/test`: 测试指令集
- `GET /api/instruction-sets/:id/statistics`: 获取统计信息

## 使用指南

### 创建指令集
1. 点击"创建指令集"按钮
2. 填写基本信息(名称、描述等)
3. 配置指令集内容和规则
4. 保存为草稿或直接激活

### 测试指令集
1. 在列表页点击"测试"按钮，或在详情页点击"测试"
2. 进入测试页面，输入测试内容
3. 选择测试模式(正常/调试)
4. 点击"开始测试"查看结果

### 管理指令集
1. 在列表页查看所有指令集
2. 使用搜索和筛选功能快速定位
3. 点击指令集名称查看详情
4. 使用操作按钮进行编辑、删除等操作

## 注意事项

1. **权限控制**: 只有授权用户才能创建和修改指令集
2. **版本管理**: 修改指令集时会自动创建新版本
3. **状态限制**: 只有活跃状态的指令集才能被执行
4. **测试环境**: 测试功能仅用于验证，不会影响生产数据
5. **性能考虑**: 大型指令集可能需要较长的执行时间

## 故障排除

### 常见问题
1. **指令集无法执行**: 检查状态是否为活跃，检查配置是否正确
2. **测试超时**: 增加超时时间设置或优化指令集性能
3. **搜索无结果**: 检查搜索关键词和筛选条件
4. **权限错误**: 联系管理员检查用户权限设置

### 性能优化
1. 合理设置指令集的复杂度
2. 定期清理无用的草稿指令集
3. 监控执行统计，优化高频使用的指令集
4. 使用分页和筛选减少数据加载量
