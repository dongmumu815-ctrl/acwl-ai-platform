# 智能体管理模块

## 概述

智能体管理模块是ACWL-AI大模型管理平台的核心功能模块，用于创建、配置和管理AI智能体。该模块支持多种类型的智能体，包括自定义聊天智能体和内容审读智能体，为用户提供灵活的AI服务。

## 功能特性

### 🤖 智能体管理
- **多类型支持**: 支持自定义(CUSTOM)和审读(REVIEW)两种智能体类型
- **状态管理**: 支持活跃、未激活等状态管理
- **可见性控制**: 支持公开和私有智能体设置
- **批量操作**: 支持批量启用、禁用、删除等操作

### 🎛️ 模型配置
- **模型选择**: 支持多种AI模型的选择和配置
- **参数调优**: 支持温度、最大令牌数等模型参数的精细调节
- **记忆功能**: 可选择启用或禁用对话记忆功能
- **提示词工程**: 支持系统提示词和用户提示词模板的自定义

### 💬 实时对话
- **即时聊天**: 提供与智能体的实时对话功能
- **消息历史**: 保存和展示完整的对话历史
- **思考过程**: 支持显示智能体的思考过程(调试模式)
- **多媒体支持**: 支持文本、图片等多种消息类型

### 🔍 内容审读
- **风险评估**: 对输入内容进行风险等级评估
- **问题检测**: 识别和标记有问题的内容片段
- **建议提供**: 为检测到的问题提供修改建议
- **置信度显示**: 显示审读结果的置信度

### 📊 统计分析
- **使用统计**: 显示智能体的总使用次数和活跃度
- **性能监控**: 监控智能体的响应时间和成功率
- **趋势分析**: 提供使用趋势和性能趋势分析

### API接口

#### 智能体管理
- `GET /api/agents`: 获取智能体列表
- `POST /api/agents`: 创建智能体
- `PUT /api/agents/:id`: 更新智能体
- `DELETE /api/agents/:id`: 删除智能体
- `GET /api/agents/:id`: 获取智能体详情

#### 对话管理
- `POST /api/agents/:id/chat`: 发送消息
- `GET /api/agents/:id/conversations`: 获取对话历史
- `DELETE /api/agents/:id/conversations`: 清空对话历史

#### 统计信息
- `GET /api/agents/statistics`: 获取整体统计
- `GET /api/agents/:id/statistics`: 获取单个智能体统计

## 使用指南

### 创建智能体
1. 点击"创建智能体"按钮
2. 选择智能体类型(自定义或审读)
3. 填写基本信息(名称、描述等)
4. 配置模型参数和提示词
5. 保存并测试智能体

### 配置提示词
1. 在编辑对话框中打开提示词编辑器
2. 编写系统提示词，定义智能体的角色和行为
3. 设置用户提示词模板(可选)
4. 使用格式化功能优化提示词结构
5. 预览和测试提示词效果

### 与智能体对话
1. 在智能体列表中点击"聊天"按钮
2. 在聊天界面输入消息
3. 查看智能体的回复和思考过程
4. 对于审读智能体，查看详细的审读结果

### 管理智能体
1. 使用搜索和筛选功能快速定位智能体
2. 查看智能体详情和使用统计
3. 根据需要编辑智能体配置
4. 管理智能体的状态和可见性

## 最佳实践

### 提示词编写
1. **明确角色定义**: 清楚地定义智能体的角色和职责
2. **设置行为边界**: 明确智能体应该做什么和不应该做什么
3. **提供示例**: 使用具体示例说明期望的回答风格
4. **使用变量**: 合理使用变量如{user_input}来引用用户输入
5. **迭代优化**: 根据实际使用效果不断优化提示词

### 模型配置
1. **温度设置**: 创意任务使用较高温度(0.7-0.9)，精确任务使用较低温度(0.1-0.3)
2. **令牌限制**: 根据应用场景合理设置最大令牌数
3. **记忆功能**: 对话型应用建议启用记忆功能
4. **模型选择**: 根据任务复杂度选择合适的模型

### 审读智能体
1. **关联指令集**: 为审读智能体关联合适的指令集
2. **风险等级**: 理解不同风险等级的含义和处理方式
3. **置信度**: 关注置信度较低的结果，可能需要人工复核
4. **建议采纳**: 合理采纳系统提供的修改建议

## 注意事项

1. **权限控制**: 确保只有授权用户能够创建和修改智能体
2. **资源消耗**: 大型模型和复杂提示词会消耗更多计算资源
3. **数据安全**: 敏感信息不应包含在提示词中
4. **版本管理**: 重要修改前建议备份智能体配置
5. **测试验证**: 新创建或修改的智能体应充分测试后再投入使用

## 故障排除

### 常见问题
1. **智能体无响应**: 检查智能体状态和模型配置
2. **回复质量差**: 优化提示词和模型参数
3. **审读结果不准确**: 检查关联的指令集配置
4. **聊天界面卡顿**: 检查网络连接和服务器状态

### 性能优化
1. 合理设置模型参数，避免过度消耗资源
2. 定期清理无用的对话历史
3. 监控智能体使用情况，优化高频使用的配置
4. 使用分页和懒加载减少页面加载时间
